# Загрузка Linux

Загрузка Linux состоит из нескольких этапов: от включения компьютера до запуска пользовательской сессии и сервисов.

Упрощенная цепочка выглядит так:

```text
BIOS/UEFI -> Bootloader -> Kernel -> initramfs -> systemd/init -> services -> login
```

## 1. Включение компьютера

После включения питание получает процессор, память и остальные устройства. Управление сначала получает прошивка материнской платы:

- `BIOS` в старых системах
- `UEFI` в современных системах

На этом этапе выполняется первичная инициализация оборудования:

- проверка памяти и процессора
- обнаружение дисков
- инициализация контроллеров
- выбор загрузочного устройства

Этот этап часто называют `POST` (`Power-On Self-Test`).

## 2. BIOS или UEFI

### BIOS

В режиме `BIOS` прошивка ищет загрузочный сектор на диске и передает управление загрузчику из `MBR` (`Master Boot Record`).

Ограничения старой схемы:

- размер диска до 2 ТБ в классическом сценарии
- ограниченное число первичных разделов

### UEFI

`UEFI` не использует старую схему `MBR` как основной механизм загрузки. Вместо этого прошивка читает файлы загрузчиков из специального раздела `EFI System Partition` (`ESP`).

Обычно там находятся `.efi` файлы, например:

```text
/boot/efi/EFI/
```

Преимущества `UEFI`:

- более современная схема загрузки
- поддержка больших дисков
- удобная работа с загрузочными записями
- поддержка `Secure Boot`

## 3. Загрузчик

После прошивки управление получает загрузчик. Чаще всего это:

- `GRUB 2`
- `systemd-boot`
- реже другие загрузчики

Задачи загрузчика:

- показать меню выбора системы
- выбрать ядро Linux
- передать ядру параметры загрузки
- загрузить `initramfs`

Для `GRUB` типичные файлы:

```text
/boot/grub2/grub.cfg
/boot/grub/grub.cfg
```

Загрузчик передает ядру параметры командной строки, например:

```text
ro quiet splash root=/dev/mapper/vg_root-lv_root
```

## 4. Загрузка ядра Linux

Ядро Linux обычно хранится в `/boot`:

```text
/boot/vmlinuz-<version>
```

Когда загрузчик запускает ядро, оно:

- распаковывается в память
- инициализирует управление памятью
- настраивает процессор
- определяет устройства
- запускает драйверы
- подготавливает монтирование корневой файловой системы

На этом этапе ядро еще не всегда может сразу получить доступ к реальному root filesystem, особенно если используются:

- `LVM`
- `RAID`
- `LUKS`
- нестандартные драйверы хранения

Поэтому нужен промежуточный этап через `initramfs`.

## 5. initramfs

`initramfs` (`initial RAM filesystem`) - это временная файловая система в памяти, которая помогает ядру подготовить настоящую корневую файловую систему.

Обычно файл находится рядом с ядром:

```text
/boot/initramfs-<version>.img
```

или

```text
/boot/initrd.img-<version>
```

Что делает `initramfs`:

- загружает нужные модули ядра
- активирует `LVM`
- собирает `RAID`
- расшифровывает диск при `LUKS`
- находит и монтирует реальный root filesystem

После этого выполняется переключение с временной корневой файловой системы на настоящую.

## 6. Корневая файловая система

Когда настоящий root filesystem найден, ядро монтирует его и запускает первый пользовательский процесс.

Исторически это был процесс `init`, а в большинстве современных систем это:

```text
/sbin/init
```

Часто эта ссылка указывает на:

```text
systemd
```

Процесс с `PID 1` имеет особое значение: он становится главным процессом пользовательского пространства.

## 7. init или systemd

Сегодня в большинстве дистрибутивов Linux используется `systemd`.

Его задачи:

- запускать системные сервисы
- монтировать файловые системы
- поднимать сеть
- запускать логирование
- управлять зависимостями между сервисами
- выводить систему в нужное состояние загрузки

Раньше часто использовались:

- `SysV init`
- `Upstart`

## 8. Targets и services в systemd

В `systemd` вместо старого понятия `runlevel` чаще используется `target`.

Примеры:

- `multi-user.target` - многопользовательский режим без графики
- `graphical.target` - многопользовательский режим с графической оболочкой
- `rescue.target` - режим восстановления

Во время загрузки `systemd` запускает `.service`, `.mount`, `.socket`, `.target` и другие unit-файлы.

Примеры полезных команд:

```bash
systemctl get-default
systemctl list-units --type=service
systemctl list-dependencies graphical.target
```

## 9. Запуск сервисов

На этом этапе запускаются системные сервисы, например:

- `udev`
- `NetworkManager`
- `sshd`
- `cron` или `crond`
- `rsyslog` или `journald`

Если настроена графическая оболочка, запускается display manager:

- `gdm`
- `sddm`
- `lightdm`

Если графики нет, система выводит приглашение входа в консоль.

## 10. Вход пользователя

Финальный этап загрузки:

- появляется консольный login prompt
- либо стартует графический экран входа

После входа пользователя запускается shell или графическая сессия.

## Как посмотреть загрузку системы

Показать сообщения текущей загрузки:

```bash
dmesg
```

Показать сообщения журнала текущей загрузки:

```bash
journalctl -b
```

Показать время загрузки:

```bash
systemd-analyze
```

Показать, какие сервисы замедляют старт:

```bash
systemd-analyze blame
```

Показать критическую цепочку загрузки:

```bash
systemd-analyze critical-chain
```

## Где смотреть важные файлы

- `/boot` - ядро, initramfs, файлы загрузчика
- `/boot/efi` - EFI-раздел в UEFI-системах
- `/etc/default/grub` - параметры GRUB
- `/etc/fstab` - монтирование файловых систем
- `/etc/systemd/system` - пользовательские unit-файлы systemd
- `/usr/lib/systemd/system` или `/lib/systemd/system` - системные unit-файлы

## Частые проблемы при загрузке

- поврежден `grub.cfg`
- неверный `UUID` в `/etc/fstab`
- отсутствует модуль в `initramfs`
- проблема с `LVM` или `RAID`
- ошибка при расшифровке диска
- зависающий сервис в `systemd`

## Короткая памятка

```text
1. BIOS/UEFI инициализирует железо
2. Прошивка передает управление загрузчику
3. Загрузчик загружает kernel и initramfs
4. Kernel запускается и ищет root filesystem
5. initramfs помогает найти и смонтировать root
6. Запускается PID 1: systemd/init
7. systemd поднимает сервисы и target
8. Появляется login prompt или GUI
```
