# TASK

Краткий список активных и запланированных задач.

## Правила

- Здесь хранится только короткое описание.
- Подробности выносятся в `TASK_XXX.md`.
- Один файл `TASK_XXX.md` соответствует одной задаче.
- Идентификатор `XXX` должен совпадать между этим списком, полным описанием и записями в `LOG.md`.

## Статусы

- `todo` - задача заведена, работа не начата.
- `in_progress` - работа идет.
- `blocked` - есть блокер, нужно решение или входные данные.
- `review` - реализация готова, нужна проверка.
- `done` - задача завершена.
